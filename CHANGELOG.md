# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- CSV/Excel formula-injection neutralisation. String cells beginning with
  `=`, `+`, `-`, `@`, `\t`, or `\r` are now prefixed with a single quote
  before being written, preventing payloads in scraped TikTok descriptions
  from executing as formulas when output files are opened in Excel /
  LibreOffice / Sheets. (`src/utils.py`, `src/storage.py`)
- Prominent legal/ethical-use block at the top of the README spelling out
  GDPR controller obligations, lawful-basis requirement, data minimisation,
  and the limits of the MIT licence.
- README "Known limitations" section listing real test-coverage numbers,
  upstream fragility, missing retry/back-off, and lack of checkpointing.

### Fixed
- **Proxy configuration is now actually applied.** Earlier versions
  silently ignored `proxy_url` in both the primary `TikTokScraper`
  (Playwright) and the `FallbackScraper` (curl_cffi), exposing the user's
  real IP to TikTok despite a configured proxy. Both scrapers now forward
  proxy settings into their respective sessions.
  (`src/scraper.py`, `src/fallback_scraper.py`)
- **`load_existing()` no longer silently drops Excel-mode files.** When
  `output_format` was set to `"excel"` the function called `pd.read_csv`
  on a `.xlsx` path, returned an empty set, and re-scraped every record
  on each run. Now reads `.xlsx` via `pd.read_excel` when appropriate.
  (`src/storage.py`)
- **TOCTOU race in `MonitorScheduler._drain` removed.** The pre-`is_new()`
  check happened outside the storage write lock and could let two
  APScheduler workers both accept the same `video_id` before either
  marked it seen. The pre-check is gone; deduplication now happens
  exactly once, atomically, inside `write_records()`. (`src/scheduler.py`)
- **`config --show` no longer leaks `proxy_username`.** The CLI's display
  table previously masked `proxy_password` but printed `proxy_username`
  in plaintext. All credential-shaped fields are now masked uniformly.
  (`main.py`)
- **Hashtag normalisation applied at the CLI boundary.** `fetch` and
  `watch` previously passed the raw user-supplied hashtag (e.g.
  `MyCats`) into storage paths while the scraper internally normalised
  it to lowercase alphanumerics, producing filename/content casing
  drift on case-sensitive filesystems. Both commands now call
  `normalize_hashtag` once, up front. (`main.py`)
- `Path(...).resolve()` applied to `--output-dir` and `--config` to
  reduce path-traversal surface when the CLI is invoked by automation.
- mypy `union-attr` warning in `src/display.py`'s `sys.stdout.reconfigure`
  fenced via an `isinstance(_, io.TextIOWrapper)` guard rather than an
  ineffective `# type: ignore[attr-defined]`.
- Spurious `nonlocal stats` declaration removed from `_run_fetch._do_scrape`
  in `main.py` — only attribute mutation occurs, no rebinding.
- One UA in the rotation pool advertised the non-existent string
  `Windows NT 11.0` (Windows 11 still reports NT 10.0). Replaced with a
  realistic Chrome 121 string. (`src/scraper.py`)

### Changed
- PyPI classifier downgraded from `Development Status :: 5 - Production/Stable`
  to `Development Status :: 4 - Beta`. Coverage on the modules that
  actually talk to TikTok is currently 0%; "Production/Stable" was
  aspirational. The README badge has been adjusted to match.
- pandas dependency relaxed from `==2.2.1` to `>=2.2.1,<4` so a clean
  install from `requirements.txt` produces an environment compatible
  with what the test suite was actually exercised against.
- Production `requirements.txt` no longer pulls in `pytest`,
  `pytest-asyncio`, or `pytest-mock`; install with
  `pip install -e ".[dev]"` for the dev tools.
- `pytest-asyncio` `asyncio_mode = "auto"` set in `pyproject.toml` to
  silence the configuration deprecation warning emitted on every run.
- `types-PyYAML` added to the `dev` optional dependency group so mypy
  can fully type-check `src/config.py`.

### Removed
- **`tenacity` and `httpx` dependencies.** Both were declared in
  `pyproject.toml` and `requirements.txt` but never imported anywhere.
  The advertised "exponential back-off on rate limits" was never
  implemented and is now removed from the README feature list rather
  than left as misleading copy.
- `docs_prompt.md` (an internal LLM prompt artefact) deleted from the
  repository root and added to `.gitignore`.

### Security
- `output/*.csv` and `output/*.xlsx` are now safe to open in Excel from
  the perspective of formula-injection attacks (see Added). Existing
  files generated by 1.0.0 should be regenerated or treated as untrusted
  spreadsheet content.

## [1.0.0] — 2026-04-17

### Added
- One-shot `fetch` command with per-hashtag progress bar and run summary.
- Continuous `watch` command using APScheduler with live Rich status table.
- `stats` command — per-file record counts, unique authors, and date range.
- `clean` command — deduplicate existing CSVs with automatic `.bak` backup.
- `config --show` / `--validate` command for merged configuration inspection.
- Dual scraper backends:
  - Primary: `TikTokApi` over Playwright Chromium.
  - Fallback: `curl_cffi` session impersonating Chrome 120 TLS fingerprint.
- 22-field `VideoRecord` schema (IDs, URLs, author, engagement, music, hashtags,
  language, region, timestamps). _Note: the original 1.0.0 release
  documented this as 23 fields; the actual count was always 22._
- CSV output with UTF-8 BOM (Excel-friendly) and optional `.xlsx` with bold
  headers, frozen panes, auto-widths, and clickable URL hyperlinks.
- Thread-safe in-memory deduplication, pre-loaded from existing output files.
- Daily file rotation — new file created when the UTC date advances.
- Layered configuration: dataclass defaults → `config.yaml` → `.env` → env vars
  → CLI flags.
- Anti-bot measures: 10 rotating User-Agents, randomised delays,
  HTTP/HTTPS/SOCKS5 proxy support. _Note: the original 1.0.0 release
  also advertised "exponential backoff on rate limits" via `tenacity`,
  but the integration was never wired up — see the Unreleased section
  for the corrective change._
- JSON Lines rotating log (10 MB × 5) + Rich console output.
- Windows cp125x terminal support via forced UTF-8 stdout and
  `legacy_windows=False`.
- 77 pytest unit and integration tests (no network calls).
